# Claude Code: Best Practices for Agentic Coding

## TL;DR & Key Conclusions
AnthropicがリリースしたClaude Codeは、開発者がClaude AIをコーディングワークフローに統合するためのコマンドラインツール。意図的に低レベルかつ意見を押し付けない設計で、特定のワークフローを強制せず、ほぼ生のモデルアクセスを提供する。本記事はClaude Codeを効果的に活用するためのベストプラクティスを体系的にまとめたもの。

- **CLAUDE.mdによるカスタマイズ**: プロジェクト固有の情報（bashコマンド、コードスタイル、テスト手順、リポジトリ慣習）をCLAUDE.mdファイルに記述することでClaudeが自動的にコンテキストとして取り込む
- **ツール権限の管理**: "Always allow"選択、`/permissions`コマンド、`.claude/settings.json`編集、`--allowedTools`フラグの4つの方法でアクセス制御可能
- **MCPサーバー連携**: Model Context Protocolサーバーを接続して機能拡張が可能
- **スラッシュコマンド**: `.claude/commands/`ディレクトリに再利用可能なコマンドを保存可能
- **Explore→Plan→Code→Commitワークフロー**: 最初にファイルを読み込み、詳細な計画を立て（"think"で拡張思考モード）、実装、最後にコミット/PRという流れが推奨
- **テスト駆動開発との親和性**: テストを先に書き、失敗を確認し、Claudeに全テストが通るまで反復実装させる
- **ビジュアル参照の活用**: デザインモックやスクリーンショットを提供することで実装精度が向上
- **Headlessモード**: `-p`フラグでCI/CD統合、Issue対応、リンティングの自動化が可能
- **マルチClaude並列実行**: 複数のClaudeインスタンスを同時実行（コード執筆と並行レビュー、別gitワークツリーでの独立タスク）
- **具体的な指示が成功率を向上**: 推論に頼らず明確に指示することで初回成功率が上がる
- **コンテキスト管理**: タスク間で`/clear`を使用して焦点を維持、Escapeで早期軌道修正

## Quickstart
1. **CLAUDE.mdファイルの作成**: リポジトリルート（または親/子ディレクトリ、ホームフォルダ）にプロジェクト情報を記述
2. **GitHub CLIのインストール**: `gh`コマンドでシームレスなGitHub連携を実現
3. **MCP サーバーの接続（オプション）**: 拡張機能が必要な場合に設定
4. **スラッシュコマンドの作成（オプション）**: `.claude/commands/`に頻繁に使うプロンプトを保存
5. **ワークフローの実践**: Explore→Plan→Code→Commitの順序で作業を進める
6. **権限設定の調整**: 必要に応じて`.claude/settings.json`でツール権限を管理

## Context & Claims (Claim-Evidence-Caveat)
- **Claim**: CLAUDE.mdファイルによるコンテキスト提供がClaude Codeの効果を大幅に向上させる
    - **Evidence**: Claudeは自動的にCLAUDE.mdの内容をプロンプトに組み込む設計
    - **Caveat**: 内容は「頻繁に使用するプロンプト」のように反復的に改善する必要がある
- **Claim**: テスト駆動開発（TDD）パターンがClaude Codeと高い親和性を持つ
    - **Evidence**: テストを先に書き、失敗を確認し、全テスト通過まで反復実装するワークフローが推奨されている
    - **Caveat**: テストが書きやすいタスクに限定される
- **Claim**: 複数Claudeインスタンスの並列実行が開発を加速する
    - **Evidence**: コード執筆とレビューの同時実行、別gitワークツリーでの独立タスク処理が可能
    - **Caveat**: 複雑なコンテキスト共有が必要な場合は注意が必要
- **Claim**: ビジュアル参照（画像、図、スクリーンショット）が実装精度を大幅に向上させる
    - **Evidence**: デザインモックを提供し、視覚的フィードバックで反復するワークフローが推奨
    - **Caveat**: テキストのみの説明では伝わりにくいUI/デザイン要件に特に有効
- **Claim**: Headlessモード（-pフラグ）でCI/CD自動化が可能
    - **Evidence**: Issue対応、リンティング、自動レビューなどのワークフロー例が紹介
    - **Caveat**: コンテナ化環境での`--dangerously-skip-permissions`使用が前提
- **Claim**: 具体的な指示が推論に頼るよりも初回成功率を向上させる
    - **Evidence**: Anthropic公式の推奨事項として記載
    - **Caveat**: タスクの複雑さや文脈によっては柔軟性とのトレードオフがある

## Normalized Conditions
- **Use Cases**:
    - 大規模なコードベースでの機能追加・リファクタリング
    - テスト駆動開発による品質担保が必要なプロジェクト
    - CI/CDパイプラインへのAI統合
    - デザインモックからのUI実装
    - 複雑なGit操作（履歴検索、コンフリクト解決、コミット管理）
    - Jupyter Notebookを使ったデータ分析の反復
    - 複数の独立したタスクを並列で進める開発
- **Anti-Cases**:
    - Claude Codeツール自体を使用しない環境
    - コマンドラインインターフェースを使用できない環境
    - 機密性が高くAIにコードを見せられないプロジェクト
    - 単発の簡単なタスク（過剰な設定コストがかかる）
- **Prerequisites**:
    - Claude Codeのインストールと設定
    - GitHub CLI（gh）の導入（GitHub連携を使う場合）
    - MCPサーバーの理解（拡張機能を使う場合）
    - コマンドライン操作の基本知識
- **Decision Triggers**:
    - 反復的なコーディング作業が多い場合
    - コードレビューやコミットメッセージ作成を効率化したい場合
    - CI/CDにAIを組み込みたい場合
    - チーム全体でAIアシスタントの使い方を標準化したい場合
    - ビジュアルデザインからの実装が頻繁にある場合
