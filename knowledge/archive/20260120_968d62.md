# Marpでスライド作成を効率化する

## TL;DR & Key Conclusions
Markdownでスライドを作成できるツール「Marp」の紹介と実践ガイド。
GUIツールのデザイン調整の手間を省き、コンテンツ作成に集中できるのが最大の特徴。

- **Markdown中心の作成**: CommonMark準拠のMarkdownでスライドを作成可能。
- **開発ツールとの親和性**: VS Code拡張機能でのプレビューや、Gitでの差分管理が容易。
- **柔軟な出力**: PDF, PPTX, HTMLへの変換をサポート。
- **カスタマイズ性**: CSSを用いたテーマ作成やスライドごとのレイアウト調整が可能。

## Quickstart
VS Codeを利用した基本的なスライド作成フロー。

1. **環境構築**: VS Code拡張機能「Marp for VS Code」をインストール。
2. **ファイル作成**: 拡張子 `.md` のファイルを作成し、先頭に `marp: true` を記述。
3. **スライド記述**: `---` でページを区切りながらMarkdownで記述。
4. **プレビュー**: VS Codeのプレビューボタンでリアルタイムに確認。
5. **エクスポート**: コマンドパレットからPDFやHTMLなどで出力。

## Context & Claims (Claim-Evidence-Caveat)
- **Claim**: スライド作成の効率化と保守性の向上
    - **Evidence**: テキストベースであるため、Gitでのバージョン管理や差分比較が容易。GUI操作による「配置の微調整」時間を削減できる。
    - **Caveat**: 高度なデザインや複雑なレイアウトを実現するにはCSSの知識が必要となり、GUIツールより直感的に操作できない場合がある。

## Normalized Conditions
- **Use Cases**:
    - エンジニアの技術発表（コードブロックが多い場合）
    - 社内ドキュメントやライトニングトーク資料の作成
    - チームでのスライド構成のレビュー（Diffで確認可能）
- **Anti-Cases**:
    - ピクセル単位での厳密なレイアウト調整が求められる場合
    - 複雑なアニメーションを多用するプレゼンテーション
    - 非エンジニアとの共同編集が多い場合（Markdown/Gitの壁）
- **Prerequisites**:
    - Markdownの基礎知識
    - VS Code (推奨エディタ)
    - CSS (カスタマイズする場合)
- **Decision Triggers**:
    - スライドのデザイン調整に時間を取られたくない時
    - スライドの内容をGitで管理したい時
- **Failure Modes / Risks**:
    - 特定のデザインを実現するためにCSSハックに時間を費やしてしまう（本末転倒）
- **Operational Cost**:
    - VS Code拡張機能の導入のみ（無料）
