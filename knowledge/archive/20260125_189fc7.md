# GitHub Copilot SDKで任意のアプリにエージェントを組み込む

## TL;DR & Key Conclusions
GitHub Copilot SDK（テクニカルプレビュー）は、GitHub Copilot CLIと同じ実証済みエージェント基盤を任意のアプリケーションに組み込めるSDKです。Node.js、Python、Go、.NETに対応し、複雑なマルチターン会話、ツールオーケストレーション、MCP統合などの煩雑な実装を抽象化します。

- **本番実績のあるエージェント基盤**: GitHub Copilot CLIで実証された実行ループをそのまま利用可能
- **マルチモデル対応**: GPT-4o、Claude、o1など複数のモデルをシームレスに切り替え可能
- **カスタムツール統合**: 独自のツール定義と実行ロジックを簡単に追加可能
- **MCPサーバー対応**: Model Context Protocolサーバーとの統合が標準でサポート
- **既存サブスク利用可能**: GitHub Copilotサブスクリプションまたは独自APIキー（BYOK）で利用可能
- **永続メモリと無制限セッション**: CLI側で導入された拡張機能により、長期コンテキストの管理が可能
- **インテリジェント圧縮**: コンテキスト制限に達する前に自動的に情報を圧縮
- **非同期タスク委任**: 重い処理をバックグラウンドで実行し、メインフローをブロックしない
- **リアルタイムストリーミング**: チャンクごとにレスポンスを受信して即座にUI反映可能
- **4言語対応**: Node.js、Python、Go、.NETで利用可能で幅広いエコシステムをカバー
- **コミュニティによる実装例**: YouTubeチャプター生成、カスタムGUI、音声ワークフロー、AIゲーム等の事例あり
- **オープンソース**: github/copilot-sdkリポジトリでセットアップ手順・サンプル・リファレンスが公開

## Quickstart
1. **インストール**: 各言語のパッケージマネージャーでSDKをインストール
   - Node.js: `npm install @github/copilot-sdk`
   - Python: `pip install github-copilot-sdk`（推定）
   - Go/.NET: 公式リポジトリの手順を参照
2. **クライアント初期化**: `CopilotClient`を作成して`start()`で起動
3. **セッション作成**: 使用するモデル（gpt-5等）を指定してセッションを生成
4. **プロンプト送信**: `session.send({ prompt: "..." })`で対話開始
5. **ストリーミング受信**: イベントハンドラーでレスポンスをリアルタイム取得（オプション）
6. **カスタムツール定義**: 独自ツールを登録してエージェントの能力を拡張
7. **MCPサーバー統合**: 必要に応じてMCPサーバーを接続してコンテキストを追加

## Context & Claims (Claim-Evidence-Caveat)
- **Claim**: GitHub Copilot SDKは、エージェントワークフロー開発の複雑性を大幅に削減する
    - **Evidence**: ターン間のコンテキスト管理、ツールオーケストレーション、モデル間ルーティング、MCPサーバー統合、パーミッション設計といった煩雑な要素をSDKが抽象化する
    - **Caveat**: テクニカルプレビュー段階であり、本番環境での安定性や機能の完全性は今後の更新に依存する

- **Claim**: GitHub Copilot CLIで実証済みの実行ループをそのまま利用できる
    - **Evidence**: SDKはCopilot CLIを支えているのと同じエージェント基盤を提供している
    - **Caveat**: CLI特有の機能（シェルとの統合等）はSDKでは別途実装が必要な場合がある

- **Claim**: 複数のAIモデルをシームレスに切り替え可能
    - **Evidence**: GPT-4o、Claude、o1などのモデルを指定して利用でき、各ステップでモデル選択可能
    - **Caveat**: モデルごとの利用料金やレート制限は別途考慮が必要

- **Claim**: カスタムツール定義により独自の機能を簡単に追加できる
    - **Evidence**: SDK APIでツールを定義・登録し、エージェントから呼び出し可能
    - **Caveat**: ツールの実装品質やエラーハンドリングは開発者に委ねられる

- **Claim**: GitHub Copilotサブスクリプションまたは独自APIキー（BYOK）で利用可能
    - **Evidence**: 既存のCopilotサブスクリプションを活用するか、独自のAPIキーを持ち込むことで利用開始できる
    - **Caveat**: サブスクリプションプランによる利用制限やAPIキーの管理責任は利用者側にある

- **Claim**: 永続メモリと無制限セッションにより長期的な対話が可能
    - **Evidence**: GitHub Copilot CLIに導入された拡張機能がSDKにも反映されている
    - **Caveat**: 「無制限」は技術的な制約内での無制限であり、実際のリソース消費には注意が必要

- **Claim**: インテリジェント圧縮により大量のコンテキストを効率的に管理できる
    - **Evidence**: コンテキスト制限に達する前に自動的に情報を圧縮する機能が提供される
    - **Caveat**: 圧縮により一部の詳細情報が失われる可能性がある

- **Claim**: 非同期タスク委任により重い処理をバックグラウンドで実行できる
    - **Evidence**: メインフローをブロックせずに処理を委任する機能がサポートされている
    - **Caveat**: 非同期処理のエラーハンドリングや結果の統合は適切に設計する必要がある

- **Claim**: リアルタイムストリーミングでレスポンスを即座にUI反映できる
    - **Evidence**: チャンクごとにレスポンスを受信するストリーミング対応
    - **Caveat**: ネットワーク遅延やクライアント側のレンダリング性能に依存する

- **Claim**: コミュニティによる多様な実装事例が存在する
    - **Evidence**: YouTubeチャプター生成、カスタムGUI、音声コマンドワークフロー、AIゲーム、要約ツール等の実装例が報告されている
    - **Caveat**: コミュニティ実装の品質や保守性は個別に評価が必要

## Code Snippets
- **基本的なセッション作成と対話**: GitHub Copilot SDKクライアントを初期化し、モデルを指定してセッションを作成し、プロンプトを送信する基本的なフロー

```typescript
import { CopilotClient } from "@github/copilot-sdk";

const client = new CopilotClient();
await client.start();
const session = await client.createSession({
    model: "gpt-5",
});

await session.send({ prompt: "Hello, world!" });
```

## Normalized Conditions
- **Use Cases**:
    - エージェント機能を自社アプリケーションに組み込みたい場合
    - マルチターン対話を含む複雑なワークフローを構築する場合
    - MCPサーバーやカスタムツールを統合したAIエージェントを開発する場合
    - GitHub Copilot CLIの実績ある基盤を独自ユースケースに活用したい場合
    - プロトタイピングから本番環境まで一貫したエージェント基盤を利用したい場合
    - 複数のAIモデルを柔軟に切り替えながら最適な応答を得たい場合
    - リアルタイムストリーミングでユーザーに即座にフィードバックを提供したい場合
    - 長期的なコンテキストを保持した対話アプリケーションを構築する場合
- **Anti-Cases**:
    - 単発のAPI呼び出しで完結する単純なAI統合（オーバーヘッドが大きい）
    - エージェントワークフローが不要なシンプルなチャットボット
    - GitHub Copilotサブスクリプションも独自APIキーも利用できない環境
    - テクニカルプレビュー段階のツールを本番環境で使用することに抵抗がある場合
    - SDKの抽象化よりも低レベルなモデルAPI制御が必要な場合
- **Decision Triggers**:
    - エージェント機能の自前実装が複雑化し、保守コストが高くなった
    - 複数のAIモデルを統合管理する必要が生じた
    - MCPサーバーやカスタムツールの統合要件が発生した
    - GitHub Copilotの実績ある基盤を活用したい意思決定がなされた
    - プロトタイプ段階でエージェント機能の検証を迅速に行いたい
    - 既存のGitHub Copilotサブスクリプションを有効活用したい
