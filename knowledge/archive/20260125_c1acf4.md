# 自社・チームのコードレビューガイドラインを作成するウィザードアプリを作りました

## TL;DR & Key Conclusions
コードレビューガイドラインをウィザード形式で作成できるWebアプリ「Code Review Guideline Creator」の紹介。約60の質問に答えるだけで、レビュアー向け・レビュイー向けのガイドラインを生成できる。

- **ガイドライン不在の課題解決**: レビュー視点の統一、属人性の排除、チーム全体の品質向上
- **柔軟な出力形式**: Markdown、PDF、LLM向けプロンプトの3形式で出力可能
- **サーバーレス設計**: SessionStorageで進捗管理、サーバー保存なしでプライバシー保護
- **AIレビュー時代への対応**: CodeRabbitなどAIコードレビューツールの取扱いについても設問を含む
- **オープンソース**: MIT Licenseで公開、カスタマイズ可能
- **実績ベース**: Googleのエンジニアリングプラクティスを基盤として構築

## Quickstart
1. **アプリへアクセス**: https://review-guideline-creator.pages.dev/ にアクセス
2. **対象選択**: レビュアー向けまたはレビュイー向けを選択
3. **質問回答**: 約60の質問に回答（SessionStorageで進捗保存）
4. **出力形式選択**: Markdown / PDF / LLM向けプロンプトから選択
5. **ガイドライン生成**: ダウンロードまたはコピーして利用

## Context & Claims (Claim-Evidence-Caveat)
- **Claim**: コードレビューガイドラインがないとレビュー品質が不安定になる
    - **Evidence**: 「レビューする際の視点が人によって違う」「気分で視点が変わってしまう」という現場の課題
    - **Caveat**: ガイドラインは作成後も定期的な見直しが必要

- **Claim**: ウィザード形式により誰でも体系的なガイドラインを作成できる
    - **Evidence**: 約60の質問に答えるだけで完成する設計、Googleのエンジニアリングプラクティスをベース
    - **Caveat**: 質問項目は一般的な内容であり、特殊な技術スタックには追加カスタマイズが必要な場合がある

- **Claim**: サーバー保存なしでプライバシーを保護できる
    - **Evidence**: SessionStorageのみを使用、サーバーへのデータ送信なし
    - **Caveat**: ブラウザのタブを閉じると進捗が失われる可能性がある

- **Claim**: AIコードレビュー時代に対応したガイドライン作成が可能
    - **Evidence**: CodeRabbitなどのAIツール取扱いについての設問を含む
    - **Caveat**: AI技術の進化に応じて設問内容の更新が必要になる可能性がある

## Normalized Conditions
- **Use Cases**:
    - チームでコードレビュープロセスを標準化したい場合
    - レビュアーとレビュイーの認識を統一したい場合
    - 新規プロジェクトでレビュー文化を構築する場合
    - AIコードレビューツール導入時のガイドライン整備
    - オープンソースプロジェクトでのコントリビューター向けガイドライン作成
- **Anti-Cases**:
    - 極めて特殊な技術スタックのみを使用するプロジェクト（質問項目が汎用的なため）
    - ガイドラインを頻繁に更新・調整したい場合（ウィザード形式のため再作成が必要）
    - 複雑な承認フローや組織固有のプロセスを含む場合（追加カスタマイズが必要）
- **Decision Triggers**:
    - コードレビューの視点がレビュアーによってバラバラになっている
    - 新メンバー加入時にレビュー方針の説明に時間がかかっている
    - AIコードレビューツールを導入したが運用方針が未整備
    - チーム規模が拡大し、レビュープロセスの標準化が必要になった
