# Claude Code 開発者自身の活用法をまとめてみた

## TL;DR & Key Conclusions
Claude Code開発者のBoris氏が実践している活用法を日本語でまとめた記事。並列実行、モデル選択、知識管理、ワークフロー最適化、品質保証、ツール統合の6つの観点から13の具体的な実践方法を紹介。最も重要なポイントは「Claudeに作業を検証する方法を与えること」で、これにより品質が2〜3倍向上する。

- **並列実行で生産性向上**: ターミナルで5つのClaudeを並行実行し、Web版とローカル版を併用して5〜10個を同時稼働させる
- **モデル選択はOpus 4.5 with thinking一択**: 大規模タスクでも修正指示が少なく、総処理時間ではSonnetより高速
- **CLAUDE.mdをチーム共有**: Claudeの失敗パターンを蓄積し、PR時に@claudeメンションでドキュメント更新
- **Planモードで計画を詰める**: 実装前にShift+Tab×2でPlanモードに入り、計画を固める
- **スラッシュコマンドで反復作業自動化**: `.claude/commands/`に保存して`/commit-push-pr`のような頻繁な操作を高速化
- **PostToolUseフックでコード自動フォーマット**: ツール実行後に自動でフォーマットをかける
- **MCPサーバー経由でツール連携**: Slack、BigQuery、Sentryなどを`.mcp.json`でチーム共有
- **検証ループが最重要**: Claudeに検証方法を与えることで品質が2〜3倍向上

## Quickstart
1. **並列実行環境の構築**: 複数ターミナルでClaude Codeを起動し、システム通知を有効化して入力タイミングを把握
2. **CLAUDE.mdの作成**: プロジェクトルートにCLAUDE.mdを配置し、Claudeの失敗パターンや注意点を記載
3. **スラッシュコマンドの設定**: `.claude/commands/`ディレクトリに頻繁に使うコマンドを定義
4. **MCPサーバーの設定**: `.mcp.json`に連携したいツール（Slack、BigQueryなど）を設定
5. **検証ループの構築**: ドメインに応じた検証方法（テストスイート、ブラウザシミュレーターなど）を整備

## Normalized Conditions
- **Use Cases**:
    - Claude Codeを日常的に使用しているチーム開発
    - 大規模なコードベースでの開発作業
    - 反復的なタスク（コミット、PR作成など）の自動化
    - 複数の外部ツール（Slack、BigQuery、Sentry）との連携が必要な場合
- **Anti-Cases**:
    - Claude Codeを初めて使う場合（基本操作の習熟が先）
    - 単発の小規模タスク
    - APIコストを厳密に管理したい場合
- **Prerequisites**:
    - Claude Codeの基本操作の理解
    - 並列実行のためのターミナル環境
    - MCPサーバー連携のための設定知識
- **Decision Triggers**:
    - 同じ作業を何度も手動で繰り返している
    - Claudeの出力品質にばらつきがある
    - チームでClaude Codeを統一的に使いたい
