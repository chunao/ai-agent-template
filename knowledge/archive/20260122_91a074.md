# Claude Code × Codex 連携入門

## TL;DR & Key Conclusions
Claude CodeとCodexをMCP（Model Context Protocol）で統合することで、それぞれの強みを活かした効率的な開発ワークフローを構築できる。Claude Codeは設計・レビュー、Codexは実装・高速処理という役割分担が効果的。

- **連携の3レベル**: 並行運用（コピペ）→差分受け渡し→MCP統合の段階的な連携方法がある
- **役割分担**: Claude Codeは要件整理・設計・タスク分解・レビュー、Codexは実装・差分作成・小規模修正を担当
- **MCP統合のメリット**: Codexをツール化してClaude Codeから直接呼び出せる
- **実践パターン**: 設計→実装→レビュー、小規模修正の連続投入、テスト先行開発など複数のワークフローが可能
- **セキュリティ注意**: 社内コードを外部LLMに渡す際は会社ポリシーに従い秘密情報をマスク処理する必要あり

## Quickstart
1. **Codex CLIのインストール**: npm経由でCodex CLIをインストール
2. **Claude Codeのインストール**: Claude Code CLIをセットアップ
3. **MCP接続**: MCPで両者を接続設定
4. **動作確認**: README追記などの簡単なタスクで連携を確認

## Normalized Conditions
- **Use Cases**:
    - 設計とレビューはClaude Code、実装はCodexと分業したい場合
    - 小規模修正を高速で連続処理したい場合
    - テスト駆動開発でCodexにテストを先行作成させたい場合
    - 2つのコーディングエージェントの強みを組み合わせたい場合
- **Anti-Cases**:
    - 単一のエージェントで完結する単純なタスク
    - セキュリティ上、外部LLMにコードを渡せない環境
    - セットアップコストをかけられない場合
- **Prerequisites**:
    - Codex CLI（npm経由でインストール）
    - Claude Code CLI
    - MCPの基本的な理解
    - 会社のセキュリティポリシーの確認
- **Decision Triggers**:
    - Claude Codeだけでは実装速度に不満があるとき
    - 設計・レビューと実装を明確に分離したいとき
    - 複数のAIエージェントを組み合わせたワークフローを構築したいとき
