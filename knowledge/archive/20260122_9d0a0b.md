# Claude Codeとghコマンドで、コードエディタをほぼ触らなくなった話

## TL;DR & Key Conclusions

Claude Code（Anthropic製CLI）とGitHub CLI（ghコマンド）を組み合わせることで、Issue駆動開発を自動化し、コードエディタをほぼ触らずに開発できる。ただし、生成AIは成果物に責任を負わないため、人間によるレビューは必須。

- **Issue駆動開発の自動化**: 「Issue#xxの開発をして」という指示だけで、コード実装からPR作成まで自動実行
- **ghコマンド活用**: GitHub CLIによりIssue/PR操作をClaude Codeから直接実行可能
- **設定ファイルでルール定義**: `.claude/CLAUDE.md`と`.claude/instruction.md`でGitHub運用ルールとブランチ命名規則を指定
- **人間レビュー必須**: 生成AIは責任を負わないため、成果物のレビューは人間が行う
- **ツールとしての活用**: 生成AIを「ツール」として使うか「エキスパート」とするかは利用者次第

## Quickstart

1. **Claude Codeのインストール**: Anthropic公式のCLIツールをインストール
2. **GitHub CLIのセットアップ**: `gh`コマンドをインストールし認証設定
3. **`.claude/CLAUDE.md`の作成**: GitHub運用ルールを定義
   - GitHub CLIの必須使用
   - Issue駆動開発アプローチ
4. **`.claude/instruction.md`の作成**: ブランチ命名規則を指定
   - Feature/Bugfix/Hotfix/Docsブランチの使い分け
5. **開発の実行**: 「Issue#xxの開発をして」と指示
6. **レビュー**: 生成されたPRを人間がレビュー

## Context & Claims (Claim-Evidence-Caveat)

- **Claim**: Claude CodeとGitHub CLIの組み合わせでコードエディタをほぼ触らなくなる
    - **Evidence**: 著者の実体験として「コードエディタの出番はありませんでした」と記述
    - **Caveat**: 初心者向けの基本的なワークフローの紹介であり、複雑なプロジェクトでの適用可能性は未検証

- **Claim**: Issue駆動開発をClaude Codeで自動化できる
    - **Evidence**: `gh`コマンドでGitHub情報取得、コード実装、テスト、PR作成、レビュー対応を自動実行
    - **Caveat**: 設定ファイル（CLAUDE.md、instruction.md）の適切な記述が前提

- **Claim**: 生成AIは成果物に対する責任を負わない
    - **Evidence**: 著者が強調している注意事項として明記
    - **Caveat**: そのため人間によるレビューが必須であり、完全な自動化は推奨されない

- **通知アラーム設定**: `~/.claude/settings.json`でmacOSの通知アラームを設定可能（長時間タスク完了時の通知用）

## Normalized Conditions

- **Use Cases**:
    - Issue駆動で開発を進めているプロジェクト
    - 定型的な実装タスクを自動化したい場合
    - コードエディタの操作を減らして効率化したい場合
    - GitHub Flowに沿った開発ワークフローを採用している場合

- **Anti-Cases**:
    - 複雑な設計判断が必要な開発
    - レビューなしで本番デプロイする運用（責任問題）
    - GitHub以外のバージョン管理システムを使用している場合

- **Decision Triggers**:
    - 繰り返しの実装タスクが多い
    - Issue管理をGitHubで行っている
    - CLIベースのワークフローを好む
    - 開発効率の向上を求めている
