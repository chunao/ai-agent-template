# 【GAS×Gemini】UI/UX意識した社内Webアプリを15分で作るプロンプト

## TL;DR & Key Conclusions

Google Apps Script（GAS）とGeminiを組み合わせて、HIG（Human Interface Guidelines）に準拠した高品質な社内Webアプリを短時間（約15分）で生成するプロンプトテンプレート。実証実験では94.5点のクオリティを達成。社内向けGoogle系ツール開発における初版・MVP作成に特化。

- **短時間で高品質**: 約1,200行のプロンプトで、30秒でタスク管理アプリ生成（Code.gs 150行 + Index.html 1,000行）
- **HIG準拠設計**: 20項目のUI/UXチェックリスト（必須7項目、フォーム8項目、UX向上5項目）を自動適用
- **GAS制約対応**: 実行時間6分上限、同時接続30人制限、非同期遅延などの制約に対応した実装パターン組み込み
- **6テーマ実装**: ライト、ダーク、オーシャン、フォレスト、サンセット、サクラの12色構成カラーパレット自動生成
- **Optimistic UI実装**: プロンプト未指示でも即座反応→サーバー送信の楽観的UI実装
- **初回体験設計**: 空状態ではなくサンプルデータ自動生成、Driver.jsによる自動ツアー起動
- **論理削除推奨**: 物理削除ではなくアーカイブ機能、4秒Undo機能付きトースト通知実装
- **適用範囲は限定的**: 社内向けGoogle系アプリ、スプレッドシートDB規模、サーバー管理不要な用途に特化
- **本番利用には追加対応必須**: アクセシビリティ、完全レスポンシブ、本番セキュリティ、パフォーマンス最適化は別途対応
- **責任線引き明確**: 生成コードはサンプル扱い、バイブコーディングでの微調整前提、配布・責任は利用者負担
- **40項目以上の品質チェック**: 自動評価シートで基本・HIG・GAS制約・細部の4レベル評価
- **実証スコア94.5%**: タスク管理アプリで200点満点中189点達成（レベル1: 100%、レベル2: 94%、レベル3: 100%、レベル4: 88%）

## Quickstart

### 1. Gemini Gemの作成
- Geminiにアクセス
- 「Gem」→「新規作成」を選択
- Gem名を「GAS Webアプリ開発アシスタント」に設定
- GitHubからプロンプト（約1,200行）をコピペして保存

### 2. アプリ生成
- チャットで「タスク管理アプリを作りたい」等の簡単な指示を送信
- 約30秒で Code.gs と Index.html が生成される

### 3. GASプロジェクトへのデプロイ
- Google Apps Scriptで新規プロジェクト作成
- Code.gs と Index.html を貼り付け
- 「デプロイ」→「ウェブアプリ」→アクセス権限設定
- デプロイURLにアクセスして動作確認

### 4. カスタマイズ（必要に応じて）
- バイブコーディング（REPL的微調整）で細部を調整
- エラーハンドリング、統計情報、フォームグループ化などを追加

## Context & Claims (Claim-Evidence-Caveat)

- **Claim**: GAS×Geminiで94.5点クオリティのWebアプリを短時間生成可能
    - **Evidence**: タスク管理アプリ生成実験で200点満点中189点達成（基本40/40、HIG Phase 1 66/70、GAS制約30/30、細部53/60）、生成時間約30秒、Code.gs 150行 + Index.html 1,000行
    - **Caveat**: 評価は著者独自基準、社内向け小規模アプリに限定、本番品質には追加対応必須

- **Claim**: HIG準拠の20項目UI/UXチェックリストで品質を自動担保
    - **Evidence**: Phase 1（必須7項目: ユーザー主導権、コンストレイント、状態視覚化、意味ある要素、具体的動詞ボタン、建設的エラー、不要確認排除）、Phase 2（フォーム8項目）、Phase 3（UX向上5項目）を自動適用
    - **Caveat**: HIGはApple/iOS向けガイドライン、Web向けには一部解釈が必要、すべての項目が100%達成されるわけではない

- **Claim**: GAS特有の制約に対応した実装パターンを組み込み
    - **Evidence**: 実行時間6分上限、同時接続30人制限、WebSocket非対応、Node.js/npm非対応の制約に対し、非同期遅延ハンドリング、処理中断不可対応、論理削除推奨、リアルタイム性限界明示を実装
    - **Caveat**: 制約対応は回避策であり根本解決ではない、大規模データや高負荷には不適

- **Claim**: Optimistic UIをプロンプト未指示でも自動実装
    - **Evidence**: タスク追加・編集・削除時に即座にUI反映→サーバー送信の楽観的UI実装、4秒Undo機能付きトースト通知
    - **Caveat**: サーバーエラー時のロールバック処理は完全ではない可能性、ネットワーク遅延時の挙動は未検証

- **Claim**: 6テーマ・12色構成のカラーパレット自動生成
    - **Evidence**: ライト、ダーク、オーシャン、フォレスト、サンセット、サクラの各テーマで背景・テキスト・UI・セマンティック色を定義、LocalStorageで設定永続化
    - **Caveat**: 色覚異常対応やWCAG準拠は未検証、カスタムテーマ追加は手動

- **Claim**: 初回体験設計でユーザーオンボーディングを自動化
    - **Evidence**: 空状態ではなくサンプルデータ自動生成、Driver.jsによる自動ツアー起動、初回起動判定でチュートリアル表示
    - **Caveat**: サンプルデータの内容は固定、ツアーのカスタマイズは手動、多言語対応なし

- **Claim**: 論理削除（アーカイブ）でデータ保全とUndo機能を両立
    - **Evidence**: 削除時に物理削除ではなくarchived=trueフラグ設定、4秒以内Undo可能なトースト通知、アーカイブ一覧で復元可能
    - **Caveat**: アーカイブデータが蓄積するとパフォーマンス低下の可能性、完全削除機能は別途実装必要

- **Claim**: 社内向けGoogle系アプリに特化、本格Webアプリには不適
    - **Evidence**: 対象ユーザー明確化（社内Google系ツール、スプシDB規模、サーバー管理不要、MVP爆速作成、GAS基礎知識あり）、非推奨ケース明示（本格Webアプリ、Google外公開、大量データ）
    - **Caveat**: 境界線が曖昧なケースあり（例: 100人規模の社内利用）、判断は利用者に委ねられる

- **Claim**: 本番利用には追加対応が必須
    - **Evidence**: アクセシビリティ（スクリーンリーダー対応）、完全レスポンシブ設計、本番レベルセキュリティ、パフォーマンス最適化が別途必要と明記
    - **Caveat**: 追加対応の具体的手順や工数見積もりは示されていない、生成コードはあくまでサンプル扱い

- **Claim**: 40項目以上の自動品質チェックリストで評価
    - **Evidence**: 4レベル評価（レベル1: 基本4項目40点、レベル2: HIG Phase 1 7項目70点、レベル3: GAS制約3項目30点、レベル4: 細部6項目60点）で総合200点満点評価
    - **Caveat**: 評価基準は著者独自、定量評価が難しい項目あり、自動評価ではなく手動確認

- **Claim**: 改善点は明確（エラーメッセージ、統計情報、フォームグループ化）
    - **Evidence**: エラーメッセージが汎用的で解決方法非提示（-3点）、統計情報（完了率など）なし（-3点）、フォーム視覚グループ化弱い（-4点）
    - **Caveat**: これらは生成後に手動追加可能、プロンプト改善で自動化できる可能性あり

- **Claim**: 責任線引きが明確で利用者負担
    - **Evidence**: 「生成コードはサンプル。バイブコーディングでの微調整が前提。配布・責任は利用者負担」と明記
    - **Caveat**: 法的責任範囲は不明確、商用利用時の責任所在は別途確認必要

## Code Snippets

- **Gemini Gem作成プロンプト**: GitHub（gas-webapp-prompt）から約1,200行のプロンプトテンプレートを取得して、Gemini Gemに設定することで、GAS Webアプリ開発アシスタントとして機能させる
- **Code.gs（約150行）**: PropertiesService連携、CRUD処理、論理削除（アーカイブ）、サンプルデータ自動生成を含むGASバックエンドコード
- **Index.html（約1,000行）**: HTML/CSS/JS統合、6テーマ機能、Driver.jsツアー、Undo付きトースト通知を含むフロントエンドコード
- **評価シート**: docs/evaluation.mdに4レベル・20項目の品質チェックリスト定義

## Normalized Conditions

- **Use Cases**:
    - 社内向けGoogle Workspace連携アプリの初版・MVP作成
    - スプレッドシートをDBとして利用可能な規模（数百〜数千行）のデータ管理
    - サーバー管理不要でデプロイ爆速な簡易Webアプリ
    - デモ前プロトタイプやたたき台の高速作成
    - GAS基礎知識がある開発者による社内ツール開発
    - UI/UX品質を一定以上担保したい小規模アプリ開発
    - 認証をGoogle任せにできる社内限定アプリ

- **Anti-Cases**:
    - 本格的なWebアプリケーション（大規模・高負荷・複雑なビジネスロジック）
    - Google外のユーザーに公開するアプリ
    - 大量データ処理（数万行以上のスプレッドシート、リアルタイム処理）
    - 実行時間が6分を超える処理
    - 同時接続30人以上の高トラフィックアプリ
    - WebSocketやリアルタイム通信が必要なアプリ
    - Node.js/npmエコシステムの活用が必要なアプリ
    - アクセシビリティ完全対応が必須なアプリ
    - 完全レスポンシブ設計が必須なアプリ
    - 本番レベルのセキュリティが必要なアプリ

- **Decision Triggers**:
    - 社内ツール開発で「とりあえず動くものを今日中に作りたい」と思ったとき
    - Google Workspaceを既に利用している組織で、追加サーバーコスト不可のとき
    - UI/UXの初期品質を一定以上担保したいが、デザイナーリソースがないとき
    - GASの基礎知識はあるが、フロントエンド実装に時間をかけたくないとき
    - MVP検証のために短期間で複数の社内ツールを試作したいとき
    - スプレッドシートDB規模（数百〜数千行）で十分なデータ量のとき
    - 同時接続ユーザーが30人以下の小規模チーム向けアプリのとき
