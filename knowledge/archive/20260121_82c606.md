# Obsidian の個人的に便利だと思う使い方

## TL;DR & Key Conclusions
Obsidian を活用することで、Markdown ベースのナレッジベースとして、Claude Code との連携、タスク管理、文献管理、実験ノート管理などを統合的に運用できる。

- **Claude Code の質問・回答を Vault に保存**: Markdown レンダリングで見やすく、デバイス間同期が可能
- **Claude Code による文書執筆・レビュー**: Vault 内のファイルを Claude Code に読み込ませて執筆や修正を依頼できる
- **カンバンボード式タスク管理**: フォルダ構造（Not Started/In Progress/Completed/Archived）でタスクを管理
- **Zotero との連携**: 添付ファイルを Vault に保存し、複数デバイスから文献 PDF を閲覧可能
- **実験ノート**: グラフや表を含む実験結果を Markdown で管理し、PDF エクスポートも可能
- **表示カスタマイズ**: CSS スニペットでファイル名のフォント調整、GitHub Alert 記法のサポート

## Quickstart
1. **Obsidian のインストール**: 公式サイトから Obsidian をダウンロードし、Vault（保管庫）を作成
2. **Claude Code 連携設定**: Claude Code に質問・回答を Vault 内に自動保存するよう設定
3. **同期設定**: Remotely Save プラグインなどで Dropbox 等の外部ストレージと同期
4. **カンバンボード作成**: Vault 内にタスク管理用のフォルダ構造を作成
5. **Zotero 連携（オプション）**: ZotMoov プラグインで添付ファイルを Vault に保存
6. **CSS スニペット設定（オプション）**: `.obsidian/snippets/custom.css` で表示をカスタマイズ

## Context & Claims (Claim-Evidence-Caveat)
- **Claim**: Claude Code の質問・回答を Vault に保存することで、Markdown レンダリングされた見やすい形で閲覧でき、デバイス間同期も可能になる
    - **Evidence**: Obsidian は Markdown を自動レンダリングし、Remotely Save などのプラグインで複数デバイス間の同期が可能。スマートフォンからも回答を確認できる
    - **Caveat**: 同期機能を利用するには外部ストレージ（Dropbox など）の設定が必要。無料版 Dropbox では 3 台までのデバイス制限がある

- **Claim**: Claude Code に Vault 内のファイルを読み込ませることで、文書の執筆・レビュー・修正を依頼できる
    - **Evidence**: Claude Code はローカルファイルを読み込んで編集可能。Obsidian の Vault は通常のファイルシステム上にあるため、Claude Code から直接アクセス可能
    - **Caveat**: MS Copilot は .md ファイルを読めないため、拡張子を .txt に変換する必要がある（その間 Obsidian から閲覧不可）

- **Claim**: フォルダベースのカンバンボード（Not Started/In Progress/Completed/Archived）でタスク管理ができる
    - **Evidence**: Obsidian 上でファイルのフォルダ間移動が容易。README.md にタスク一覧と締切を記述し、Python スクリプトで整合性チェックや警告も可能
    - **Caveat**: 複雑なタスク管理には専用ツール（Jira、Trello など）の方が適している場合もある

- **Claim**: Zotero の添付ファイルを Vault に保存することで、複数デバイスから文献 PDF を閲覧できる
    - **Evidence**: ZotMoov プラグインで添付ファイルを Vault に移動可能。Obsidian は PDF も認識・表示するため、Zotero がインストールされていないデバイスからも閲覧可能
    - **Caveat**: Zotero と Dropbox の容量管理が必要。大量の PDF を保存すると容量を圧迫する

- **Claim**: 実験ノートとして、グラフや表を含む実験結果を Markdown で管理できる
    - **Evidence**: Obsidian は画像（PNG、JPEG など）や表（HTML、Markdown 形式）を表示可能。pandas の `to_html()` や `to_markdown()` で DataFrame を直接貼り付けられる
    - **Caveat**: HTML ノート出力より実装コストは低いが、複雑なインタラクティブな可視化には向いていない

- **Claim**: CSS スニペットでファイル名のフォント調整や、GitHub Alert 記法のサポートが可能
    - **Evidence**: `.obsidian/snippets/custom.css` に CSS を記述することで、ファイル名の色やサイズをカスタマイズ可能。GitHub Alert 記法（`[!Note]`、`[!Tip]` など）にも対応
    - **Caveat**: CSS の知識が必要。過度なカスタマイズはメンテナンスコストを増やす可能性がある

## Code Snippets
- **ファイル名のフォントカスタマイズ CSS**: `.obsidian/snippets/custom.css` に以下を記述してファイル名のフォントを薄く小さくする
- **タスク整合性チェック Python スクリプト**: README.md とタスクファイルの整合性をチェックし、締切警告を出す
- **拡張子一斉変換スクリプト**: .md ファイルを .txt に一斉変換（MS Copilot 用）

## Normalized Conditions
- **Use Cases**:
    - Claude Code との会話履歴を保存・管理したい場合
    - 個人用のタスク管理システムを軽量に構築したい場合
    - Zotero で管理している文献 PDF を複数デバイスから閲覧したい場合
    - 実験結果をグラフや表を含めて Markdown で記録したい場合
    - Markdown ベースのナレッジベースを構築したい場合
- **Anti-Cases**:
    - リアルタイム共同編集が必要な場合（Notion、Google Docs の方が適切）
    - 複雑なプロジェクト管理が必要な場合（Jira、Asana など専用ツールが適切）
    - インタラクティブな可視化が必要な実験ノート（Jupyter Notebook の方が適切）
    - Markdown に馴染みがないユーザー
- **Prerequisites**:
    - Obsidian のインストール
    - Claude Code（文書レビュー連携を行う場合）
    - Remotely Save プラグイン（デバイス間同期を行う場合）
    - Dropbox などの外部ストレージ（同期を行う場合）
    - Zotero と ZotMoov プラグイン（文献管理連携を行う場合）
    - CSS の基礎知識（表示カスタマイズを行う場合）
- **Decision Triggers**:
    - Claude Code の会話履歴を体系的に管理したくなった時
    - GitHub Issues などの大規模なタスク管理システムが過剰と感じた時
    - 文献 PDF を複数デバイスから閲覧したくなった時
    - 実験結果を Markdown で管理したくなった時
- **Failure Modes / Risks**:
    - 同期設定を誤ると、デバイス間でファイルが同期されない
    - Dropbox の容量制限により、大量の PDF や画像を保存できない可能性
    - CSS カスタマイズの過度な使用によるメンテナンスコストの増加
    - MS Copilot 連携時の拡張子変換による運用の複雑化
- **Operational Cost**:
    - 初期セットアップ時間（Obsidian インストール、プラグイン設定、同期設定など）
    - 外部ストレージの利用料（Dropbox 有料版など、3 台以上のデバイス同期時）
    - CSS スニペットのメンテナンス時間
    - Python スクリプト作成・保守の時間（タスク管理自動化を行う場合）
