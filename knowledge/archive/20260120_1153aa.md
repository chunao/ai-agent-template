# Google AI Proで使い倒すAntigravity開発ノウハウ：モデルとモードの最適解

## TL;DR & Key Conclusions
Antigravity IDE での効率的な開発には、タスクの性質に合わせたモデル選択、実行モードの切り替え、プロジェクト固有のナレッジ共有が重要。本記事は、Google AI Pro環境でのAntigravity活用における実践的なベストプラクティスを提供する。

- **モデル選択の戦略性**: Gemini (Low)は軽微な修正に、Sonnet 4.5 (Thinking)は複雑なロジックに使い分ける
- **Planningモード優先**: 作業の9割でImplementation Planを先に作成し、大規模破壊を防ぐ
- **Fastモードの活用**: 単一ファイルの微調整や自明なバグ修正では計画をスキップしてスピード重視
- **AGENTS.mdの必須性**: プロジェクトルートに配置することで、コーディング規約や検証コマンドをAIに注入
- **自律型ワークフロー**: ブランチ作成からPR作成まで、定型化された指示で一任可能
- **設計時間の確保**: 実装の自動化により、仕様検討や設計レビューにリソースを集中できる

## Quickstart
1. **モデル選択**: 軽微な修正にはGemini (Low)、複雑な処理にはSonnet 4.5 (Thinking)を選択
2. **モード設定**: 大規模変更はPlanningモード、単一ファイル修正はFastモードを選択
3. **AGENTS.md作成**: プロジェクトルートに配置し、技術スタック、コーディング規約、検証コマンド、禁止事項を記述
4. **自律ワークフロー実行**: 定型プロンプトでブランチ作成→計画提示→実装→検証→コミット→PR作成を一任
5. **レビューポイント確認**: Implementation Planを確認し、必要に応じてフィードバック

## Context & Claims (Claim-Evidence-Caveat)
- **Claim**: Planningモードを作業の9割で使用すべき
    - **Evidence**: Implementation Planを事前作成することで、大規模な破壊を防ぎ、実装前に人間が設計をレビューできる
    - **Caveat**: 単一ファイルの微調整やドキュメント更新など自明な作業では、計画作成のオーバーヘッドがスピードを損なう
- **Claim**: AGENTS.mdは「プロジェクトらしい」コードを生成するための必須要素
    - **Evidence**: 技術スタック、コーディング規約、検証コマンド、禁止事項を定義することで、毎回同じ指示を繰り返す必要がなくなる
    - **Caveat**: AGENTS.mdの内容が古くなったり不正確な場合、AIが誤った判断をする可能性がある
- **Claim**: 自律型ワークフローにより、ブランチ作成からPR作成まで一任できる
    - **Evidence**: 「手順」と「品質基準」をセットで伝えることで、エージェントは自律的にターミナルを叩き、検証を行いながらPRまで辿り着く
    - **Caveat**: 複雑な依存関係や環境固有の問題がある場合、途中で人間の介入が必要になる可能性がある
- **Claim**: モデルの使い分けが効率と精度の両立に直結
    - **Evidence**: Gemini (Low)は速度優先の軽微な修正に、Sonnet 4.5 (Thinking)は複雑なロジックやリファクタリングに最適化されている
    - **Caveat**: タスクの複雑度の判断を誤ると、低性能モデルでは品質が低下し、高性能モデルでは時間とコストが無駄になる

## Code Snippets
- **汎用ワークフロープロンプト**: ブランチ作成→計画提示→実装と検証→コミット→PR作成の一連のフローを一任するための定型プロンプト。Conventional Commits形式でのコミットメッセージ記述を含む
- **AGENTS.md設定例**: 技術スタック、コーディング規約、検証コマンド、禁止事項、PRテンプレートを定義するためのサンプル構成


## Normalized Conditions
- **Use Cases**:
    - 複数ファイルにまたがる新機能の実装
    - 大規模なリファクタリング作業
    - プロジェクト固有の規約に従った自動化開発
    - ブランチ作成からPR作成までの定型作業の一任
    - Git操作を含む自律型ワークフローの構築
- **Anti-Cases**:
    - 単一ファイルの微調整（Fastモードで十分）
    - ドキュメントのみの更新
    - プロジェクト規約が未定義または頻繁に変更される状況
    - 環境固有の複雑な依存関係がある場合（人間の介入が必要）
- **Prerequisites**:
    - Antigravity IDEのインストールと設定
    - Google AI Proのアカウント
    - プロジェクトルートへのAGENTS.md配置（推奨）
    - Gitリポジトリの初期化
    - 基本的なGit操作の理解
- **Decision Triggers**:
    - 複数ファイルにまたがる修正が必要になった
    - 実装前に設計レビューが必要な大規模変更
    - 同じ指示を何度も繰り返している
    - PR作成までの定型作業に時間がかかっている
    - チーム開発でコーディング規約を統一したい
- **Failure Modes / Risks**:
    - AGENTS.mdの内容が古くなり、AIが誤った判断をする
    - タスクの複雑度判断を誤り、不適切なモデルやモードを選択
    - 自律ワークフローが環境固有の問題で途中停止
    - Implementation Planのレビューを怠り、設計ミスが実装に反映
- **Operational Cost**:
    - AGENTS.mdの作成と定期的な更新のコスト
    - Implementation Planのレビュー時間
    - モデル使用料（Sonnet 4.5 Thinkingは高コスト）
    - Git操作やCI/CDパイプラインの実行時間
