# Antigravityのスキルで、実装・レビュー・修正まで一貫してできるようになった話

## TL;DR & Key Conclusions
GoogleのIDE「Antigravity」において「スキル」という新機能が追加され、一連の定型作業をAIに学習させてワークフローを自動化できるようになった。本記事では、レビュー後の修正対応を自律的に行うスキルを構築し、プルリクのURLからレビュー内容を確認して修正コミットまで一貫してAIに実行させることに成功した事例が紹介されている。

- **スキルの定義場所**: プロジェクト固有のスキルは`.agent/skills/(任意のディレクトリ)/SKILL.md`、共通スキルは`~/.gemini/antigravity/skills/(任意のディレクトリ)/SKILL.md`に配置する
- **スキルの自動生成**: SKILL.mdファイルはAntigravity自身に作成させることが最も効率的
- **レビュー修正の流れ**: Gemini Code Assistでレビュー後、ghコマンドやChrome DevTools MCPでレビュー内容を取得し、修正方針を確認させてコミットまで実行
- **実行結果**: ゼロ除算エラーのレビュー指摘に対し、AIが自動的に例外チェック処理を追加して修正コミットを作成
- **Devinとの比較優位性**: 月額固定でコストパフォーマンスが高い、自分のアカウントでコミットするため責任感を維持できる、スキルによる柔軟な制御が可能
- **応用範囲の広さ**: レビュー修正のほか、テストコード自動生成やドキュメント更新など様々なルーチンワークに応用可能

## Quickstart
1. **プロジェクトディレクトリを作成**: `.agent/skills/`配下に任意のディレクトリを作成し、その中に`SKILL.md`ファイルを配置する
2. **スキル定義を作成**: Antigravityに「（行いたい処理）を行うスキル用のファイルを作成して」と依頼し、SKILL.mdを自動生成させる
3. **必要なツールを準備**: レビュー確認のため、ghコマンドやChrome DevTools MCPを有効化しておく
4. **スキルを実行**: 「（プルリクURL）をレビューして修正して」とAntigravityに指示し、自律的なレビュー修正フローを実行させる
5. **結果を確認**: 修正コミットが自動的に作成されたことを確認する

## Context & Claims (Claim-Evidence-Caveat)
- **Claim**: Antigravityのスキル機能により、レビュー修正という一連の作業を完全に自動化できる
    - **Evidence**: プルリクのURLを渡すだけで、AIがレビュー内容を取得し、修正方針を確認してコミットまで実行した実例が示されている
    - **Caveat**: ghコマンドの設定が不十分だとブラウザ経由での間接的な操作になる可能性がある。また、スキルの定義内容によって実行精度が変わる

- **Claim**: SKILL.mdファイルはAntigravity自身に作成させるのが最も簡単
    - **Evidence**: 「行いたい簡単な命令を行うスキル用のファイルを作成して」と依頼することで、AIが自動的にスキルファイルを生成した
    - **Caveat**: 2回試行したところ2回目のディレクトリ構造が想定と異なったため、生成されたファイルの内容や構造の確認は必須

- **Claim**: Devinと比較してAntigravityの方がコスト面と制御面で優位性がある
    - **Evidence**: Geminiの月額利用料のみで使用でき、自分のアカウントでコミットされるため責任感が維持される。スキルによって明示的に手順を定義できるため勝手な挙動を防げる
    - **Caveat**: Devinとの直接的なベンチマーク比較データは示されておらず、あくまで著者の主観的な評価

- **Claim**: スキルによってAIの誤操作を防ぎ、リソース消費を抑えられる
    - **Evidence**: 事前に手順を定義することで、AIが一定の方向性を持って動作するため、不要な試行錯誤が減る
    - **Caveat**: スキル定義の品質に依存し、不適切な定義では逆に非効率になる可能性がある

## Normalized Conditions
- **Use Cases**:
    - レビュー後の修正を自動化したい開発者
    - 定型的なコード修正やレビュー対応を繰り返し行う開発チーム
    - ゼロ除算エラーなどの典型的なバグへの対応を自動化したい場合
    - テストコードやドキュメント更新などのルーチンワークを効率化したい場合
    - 複数プロジェクトで共通の作業フローを統一的に管理したい場合
- **Anti-Cases**:
    - レビュー内容が高度に文脈依存的で、AIによる判断が難しい修正
    - 人間の創造性や設計判断が強く求められる開発作業
    - スキル定義を作成・管理するコストが修正作業の自動化メリットを上回る場合
    - AIの自律的な動作に対する監視・検証体制が整っていない環境
- **Prerequisites**:
    - Antigravity IDEのセットアップ
    - Gemini Code Assistの有効化
    - ghコマンドの導入と設定（推奨）
    - Chrome DevTools MCPの有効化（レビュー内容取得用）
    - `.agent/skills/`ディレクトリ構造の理解
    - YAML形式とMarkdown形式の基本知識
- **Decision Triggers**:
    - レビュー後の修正作業が定型的で頻繁に発生する場合
    - 開発チームが複数プロジェクトで同一のレビュー修正フローを運用している場合
    - AIによる自動修正の精度が十分に検証され、実用レベルに達した場合
    - 人的リソースが不足しており、定型作業の自動化が急務となった場合
- **Failure Modes / Risks**:
    - スキル定義が不十分だと、AIが意図しない動作をする可能性
    - ghコマンド設定が不完全だと、ブラウザ経由の間接的操作になり効率が低下する
    - レビュー内容の解釈ミスによる不適切な修正が行われるリスク
    - AIが生成するコードの品質担保が不十分で、新たなバグを生む可能性
    - スキルファイルの自動生成時にディレクトリ構造が想定外になることがある
- **Operational Cost**:
    - Geminiの月額利用料（従量課金ではないためコスト予測が容易）
    - SKILL.mdファイルの作成・保守にかかる時間（初回は1時間程度）
    - レビュー修正結果の検証にかかる時間
    - スキル定義の継続的な改善・調整作業
