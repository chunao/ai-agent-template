# Claude Codeで「外部脳」を構築する—情報過多時代のナレッジマネジメント実践

## TL;DR & Key Conclusions
Claude CodeのSubagentとSlash Command機能を活用し、情報を「貯める」だけでなく「使える形」に自動加工するナレッジマネジメント手法の解説。Building a Second BrainのCODEフレームワークをベースに、Markdown設定ファイルでAIの「型」を定義することで、質の高い「自分専用のWikipedia」を構築する。

- **型による品質担保**: SubagentとSlash CommandでAIに明確な制約（推測禁止、リンク必須など）を与えることで、出力の安定性を確保。
- **定型と対話の融合**: 議事録整理などの定型処理を自動化しつつ、その後のメール作成などの非定型タスクを自然言語で実行可能。
- **ローカル管理の優位性**: データはGit管理可能なMarkdownファイルとしてローカルに保存され、永続的な知識ネットワークとして成長する。

## Quickstart
Claude Code環境で「外部脳」を構築する手順。

1. **ディレクトリ作成**: プロジェクトルートに `.claude/commands` と `.claude/agents` を作成。
2. **Subagent定義**: `.claude/agents/note-distiller.md` などを作成し、役割・ツール・制約（1ノート1アイデア等）を記述。
3. **Command定義**: `.claude/commands/distill.md` などを作成し、トリガーと処理フローを記述。
4. **実行**: `/distill inbox/meeting.md` で整理、`/ask` で検索・回答生成を実行。

## Context & Claims (Claim-Evidence-Caveat)
- **Claim**: 情報は整理コストが高いため、貯まるだけで使えない状態になりがちである。
    - **Evidence**: ビジネスパーソンが「あの件いつ決まった？」に即答できず、同じ調査を繰り返している現実。
    - **Caveat**: Building a Second Brainなどの既存手法は有効だが、手動での継続的な実践は手間がかかりすぎる。

- **Claim**: AI（Claude Code）に「型」を与えることで、実用的なナレッジベース構築が可能になる。
    - **Evidence**: 「推測で埋めない」「根拠リンク必須」といった制約を明文化することで、ハルシネーションを防ぎ信頼性を担保できる。
    - **Caveat**: 抽象的な指示（「良い感じに」）ではなく、具体的なルール記述が必要不可欠。

- **Claim**: NotebookLMとは異なる「自分専用のWikipedia」構築に適している。
    - **Evidence**: NotebookLMは都度解釈の検索エンジンだが、本手法は情報の構造化とリンク結合により知識ネットワークを永続化する。
    - **Caveat**: 一時的な調査にはNotebookLMの方が手軽で向いている。

## Normalized Conditions
- **Use Cases**:
    - 長期的なプロジェクトでの知識蓄積と経緯追跡
    - チーム内での厳密な仕様管理や意思決定の共有
    - 定型的な議事録整理と、それに続く報告業務の効率化
- **Anti-Cases**:
    - 一時的な調査や、使い捨ての質問
    - セットアップの手間をかけたくない短期的なタスク
- **Prerequisites**:
    - Claude Codeの利用環境
    - Gitによるファイル管理の基礎知識（推奨）
- **Decision Triggers**:
    - 情報過多により重要事項の検索に時間がかかっている場合
    - 「なぜそう決めたか」という経緯の喪失がリスクになっている場合
- **Failure Modes**:
    - 抽象的な指示によりAIの出力品質がブレる
    - Subagentに過剰な権限を与えてしまい意図しない動作をする
- **Operational Cost**:
    - 初期設定（ディレクトリ構成やプロンプト作成）の工数
    - 運用に伴う設定ファイルの微調整とメンテナンス
