# Claude Code×mdファイルで挫折しないAI時代のタスク管理術

## TL;DR & Key Conclusions
Notion、Asana、Trelloなどのタスク管理アプリに挫折した人向けに、プレーンなMarkdownファイル（weekly.md）とClaude Codeを組み合わせたシンプルなタスク管理手法を紹介。著者が2週間運用した結果、タスク消化率が約2倍に向上した。

- **mdファイル + Claude Codeの組み合わせ**: タグやプロパティの設計が不要で、テキストファイルとAIだけで運用できるため、心理的ハードルが低く続けやすい
- **複数ファイルを横断した照合が可能**: 戦略ドキュメントや計画ファイルと照らし合わせて、矛盾や抜け漏れを自動検出してくれる
- **常時表示と低操作コスト**: ターミナルベースのClaude Codeなら画面の隅に常時表示でき、UIの階層も少ないため即座にアクセス可能
- **「詰まり解消」のスキル化**: /unblockスキルで神経科学ベースのフレームワーク（Affective Labeling + Head/Heart/Hand診断 + If-Thenプラン）を実装し、先延ばしを解消
- **週次レビューの自動化**: /weeklyスキルで週の振り返りと次週計画を自動化、戦略との整合性も確認可能
- **タスク消化率が2倍に**: 2〜3週間の運用で従来比約2倍のタスク消化を達成し、「何をするか」で悩む時間が明らかに減少
- **チーム同期には不向き**: 個人のタスク管理には優れているが、チームでの状況共有にはNotionなどの方が適している
- **意志の弱さには無力**: ツール自体はやる気を生まないが、「何が詰まっているか」を言語化する問いかけによって自己認識は高まる
- **完璧ではないが「比較的うまくいっている」感覚**: Notion時代と比較して挫折せず継続できている実感がある
- **スキルパッケージを配布**: weekly.mdテンプレート、/weekly、/unblock、/setupの各スキルを含むパッケージを提供
- **Obsidianをビューアーとして使用**: mdファイルをObsidianで開くことで、余計な機能がなく心理的ハードルが低い閲覧体験を実現
- **GPT-Proとの併用**: Claude Codeがまとめた戦略ファイル群をGPT-Proに渡してレビューを受け、その結果を再度Claudeに提供することで精度向上

## Quickstart
1. **すべてのToDo管理アプリを捨てる**: Notion、Asana、Trelloなどの複雑なツールを一旦手放す
2. **weekly.mdファイルを作成**: プレーンなMarkdownファイルとして週次タスク管理ファイルを作成（記事内のテンプレートを参考）
3. **Obsidianで開く**: weekly.mdをObsidianで開いてビューアーとして使用（シンプルで心理的ハードルが低い）
4. **Claude Codeを隣に開く**: ターミナルでClaude Codeを起動し、画面の隅に常時表示
5. **スキルをインストール**: /setup、/unblock、/weeklyの各スキルをインストール（記事内でパッケージ配布）
6. **日々の使い方**:
   - タスクが山積みで動けないときは `/unblock` を実行して優先順位整理
   - 週の振り返りと次週計画には `/weekly` を実行
   - 「今週何が残ってる？」など自然言語でClaude Codeに問いかける
7. **戦略ドキュメントとの照合**: 年次計画や月次計画ファイルをClaude Codeに読ませて、weekly.mdとの整合性を確認

## Context & Claims (Claim-Evidence-Caveat)
- **Claim**: NotionやAsanaなどの高機能タスク管理アプリは設計の複雑さから挫折しやすい
    - **Evidence**: 著者はNotion、Asana、Trelloを使ったが、タグやプロパティの設計、ビューの作り込みに時間を取られ、2週間程度で入力が面倒になり放置が始まるパターンを繰り返した
    - **Caveat**: 個人の運用スタイルや好みによっては、高機能ツールが合う人もいる。特にチーム運用ではNotionのような共有機能が有効な場合もある

- **Claim**: mdファイル + Claude Codeの組み合わせは、シンプルで続けやすいタスク管理を実現できる
    - **Evidence**: 著者がweekly.mdというMarkdownファイル1つで運用を開始し、2〜3週間でタスク消化率が従来比約2倍になり、weekly.mdは毎日使うようになった
    - **Caveat**: この手法は個人のタスク管理に特化しており、チーム同期や複数人での状況共有にはNotionの方が適している

- **Claim**: ターミナル版のClaude CodeはCoworkやChatGPTよりもタスク管理に向いている
    - **Evidence**: ターミナルなら(1)画面の隅に常時表示可能、(2)UIがシンプルでボタン操作不要、(3)階層構造がなく「このファイルを見て」で一発アクセス可能、という3つの利点がある
    - **Caveat**: ターミナル操作に不慣れな非エンジニアにとっては、CoworkなどのGUI版の方が使いやすい場合もある

- **Claim**: Claude Codeは複数ファイルを横断して矛盾や抜け漏れを検出できる
    - **Evidence**: 著者が年次計画・月次計画とweekly.mdを同時にClaude Codeに読ませたところ、日付の間違い（1/22は木曜日なのに水曜と記載）や月末加入の問題点を指摘された
    - **Caveat**: AIの指摘が常に正確とは限らず、最終的な判断は人間が行う必要がある

- **Claim**: /unblockスキルは神経科学に基づいたフレームワークで詰まり解消に有効
    - **Evidence**: ある神経科学者のフレームワーク（Affective Labeling → Head/Heart/Hand診断 → If-Thenプラン）を元に実装し、著者が実際に先延ばしから脱出できた実例がある
    - **Caveat**: 意志の弱さが根本原因の場合、どんなツールやフレームワークも完全には解決できない

- **Claim**: AIには弱さを見せやすく、上司やチームには言いづらいことも相談できる
    - **Evidence**: 著者が「パンクしそう」「寝落ちしてしまった」などの状況をClaude Codeに伝え、即座に整理とTODO更新を受けられた
    - **Caveat**: AIに依存しすぎると、人間との対話やチーム内でのコミュニケーションスキルが低下する可能性がある

- **Claim**: Claude CodeとGPT-Proを併用することで、より精度の高い計画レビューが可能
    - **Evidence**: Claude Codeが戦略ファイルをZIPにまとめてGPT-Proに提供し、GPT-Proのフィードバックを再度Claudeに渡すことで、日付エラーや月末加入問題などを発見できた
    - **Caveat**: 複数のAIプラットフォームを併用するため、操作の手間やコストが増える

- **Claim**: mdファイル形式は他のプラットフォームへの展開が容易
    - **Evidence**: プレーンテキストのMarkdownファイルであるため、コーディングエージェントが容易に読み込め、複雑なボタン操作も不要
    - **Caveat**: 記事内では具体的な他プラットフォームへの展開例は示されておらず、実際の移行の容易さは検証されていない

## Code Snippets
- **weekly.mdテンプレート**: 週次タスク管理用のMarkdownフォーマット。「今週のフォーカス」「曜日ごとのタスク（チェックボックス形式）」「優先度マーカー（★最優先）」を含む構造で、Obsidianでの表示に最適化されている
- **/unblockスキル**: 詰まったときに次の一手を出すためのスキル。(1)今の気分を1語で聞く（Affective Labeling）、(2)Head/Heart/Handで何が詰まっているかを診断、(3)If-Thenプランを作成して再開、という3ステップを自動実行
- **/weeklyスキル**: 週の振り返りと次週計画を自動化するスキル。戦略ドキュメントとの照合機能も含まれ、計画との矛盾を検出可能
- **/setupスキル**: 初回セットアップをClaude Codeが案内してくれるスキル。インストール後の初期設定を対話形式でサポート

## Normalized Conditions
- **Use Cases**:
    - Notion、Asana、Trelloなどの高機能タスク管理アプリに何度も挫折した経験がある
    - プロパティやタグの設計で手が止まってしまい、タスク管理ツールが続かない
    - タスクが山積みで「何から手を付ければいいか」わからず動けない状態が頻繁にある
    - 個人のタスク管理をシンプルに運用したい
    - 複数の計画ファイル（年次計画、月次計画など）との整合性をチェックしたい
    - ターミナル操作に抵抗がなく、常時表示可能なツールを求めている
    - AIに弱さや悩みを率直に相談したい
    - 「詰まり」や先延ばしの原因を言語化して解消したい
- **Anti-Cases**:
    - チームでのタスク管理や状況共有を重視する場合（NotionやAsanaの方が適している）
    - 非エンジニアでターミナル操作に不慣れな場合（CoworkやChatGPTのGUI版の方が使いやすい）
    - プロパティやタグでの厳密な分類・検索が必要な高度なタスク管理を求める場合
    - 意志の弱さが根本原因でツール以前の問題がある場合
    - 複数のAIプラットフォーム（Claude Code + GPT-Pro）を併用する手間をかけたくない場合
- **Prerequisites**:
    - Claude Codeがインストールされていること
    - Obsidian（またはMarkdownエディタ）がインストールされていること
    - ターミナル操作の基本的な知識
    - mdファイルを保存・管理できる環境
    - （オプション）戦略ドキュメントや計画ファイルがmd形式で存在すること
    - （オプション）GPT-Proアカウント（Claude Codeとの併用を行う場合）
- **Decision Triggers**:
    - タスク管理アプリを開くのが億劫になり、放置期間が2週間を超えた
    - プロパティ設計の段階で手が止まり、正解がわからないまま進めなくなった
    - 「何をするか」で悩む時間が実際の作業時間よりも長い
    - タスクが多すぎてパンク寸前で、優先順位を整理してほしい
    - 計画と実際のTODOに矛盾があると感じるが、自分では見つけられない
    - AIに率直に相談して、問いかけを受けながら整理したい
