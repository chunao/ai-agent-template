# "動くけど、これじゃない"からの脱却 - 古参エンジニアがAntigravityとの対話で見つけたこと

## TL;DR & Key Conclusions

古参エンジニアがGoogleのAIコーディングエージェント「Antigravity」を使い、段階的なプロンプト改善を通じてプロダクションレディなコードを目指した実験レポート。3段階のPhaseで「機能実装」から「品質基準の明示」「エンジニアリング原則の言語化」へと進化させた。

- **AIへのプロンプトは要件定義能力の鏡**: AIは指示されたことを完璧に実行するが、推測や忖度はしない。不満足な結果は指示側の言葉足らずが原因
- **"常識"の言語化が競争優位性**: エンジニアが培った「テストの必要性」「ユーザーのつまずきポイント」などの勘所を明示的に伝える力がAI時代の武器になる
- **スコープの明確化が重要**: AIはスコープ内では完璧に機能するが、暗黙の前提は共有されない
- **Phase 1（機能優先）の問題**: テストコードなし、英語の例外メッセージ、ユーザーフレンドリーでない
- **Phase 2（品質追加）の気づき**: テスト対象が本番コードではなく抽出ロジック、言語指定の不足で英語出力に変化
- **Phase 3（完全要件）の成果**: 日本語対応、親切なエラーメッセージ、本番コードのテスト、カバレッジ出力を明示してプロダクションレディな成果物を実現
- **Antigravityの特性**: すごく頼もしい職人。ただしアタマの中の「常識」は丁寧に共有すべし
- **エンジニアの経験価値**: AIが普及する時代こそ、経験で培った「こだわり」や「勘所」がペアプログラミングの質を左右する

## Context & Claims (Claim-Evidence-Caveat)

- **Claim**: AIへのプロンプトは、そのまま自分の要件定義能力の鏡である
  - **Evidence**: Phase 1で「機能優先」の指示のみを出した結果、テストコードなし・英語エラーメッセージ・ユーザー案内なしのコードが生成された。Phase 3で品質要件を明示した結果、プロダクションレディなコードが生成された
  - **Caveat**: この結論はAntigravity使用時の実験であり、他のAIツールでは特性が異なる可能性がある

- **Claim**: AIは「常識」や「空気を読む」ことができない
  - **Evidence**: Phase 2で「日本語で」と明示しなかったため、テストコードやメッセージがすべて英語に変わった。日本人向けツールを作るという前提は共有されなかった
  - **Caveat**: 明示的な指示を与えれば、AIはその範囲内で完璧に機能する

- **Claim**: テスト対象のスコープを明確にしないと、本番コードがテストされない
  - **Evidence**: Phase 2で「全パスをテストする」と指示したが、Antigravityはロジックを別ファイルに抽出し、そのロジックのみをテストした。本番環境で実行されるコード自体はテストされていなかった
  - **Caveat**: Phase 3で「リリースするコードそのものをテストすること」と明示した結果、本番コードがテストされるようになった

- **Claim**: エンジニアの「常識」がAI時代の最大の武器になる
  - **Evidence**: Phase 3で「エラーメッセージには『何が起こったのか』に加えて『ユーザはどうしたらよいか』を表現すること」という経験則を言語化した結果、ユーザーフレンドリーなエラーハンドリングが実装された
  - **Caveat**: これらの「常識」を言語化して伝える能力が必要。暗黙知のままではAIは活用できない

- **Claim**: Antigravityは「すごく頼もしい職人」
  - **Evidence**: 明確な指示を与えれば、期待通りのプロダクションレディなコードを生成できた
  - **Caveat**: アタマの中の「常識」は丁寧に共有する必要がある

## Normalized Conditions

- **Use Cases**:
  - AIコーディングエージェントを使って初めて開発を行うエンジニア
  - プロンプトエンジニアリングのベストプラクティスを学びたい場合
  - AI駆動型開発で品質基準を満たす成果物を得たい場合
  - 要件定義スキルをAI活用の文脈で向上させたい場合
  - エンジニアリング経験をAI時代にどう活かすか模索している古参エンジニア

- **Anti-Cases**:
  - すでにAI駆動開発の経験が豊富で、プロンプト設計のコツを熟知している場合
  - 単純なプロトタイピングで品質要件が低い場合
  - Antigravity以外のツール（Claude Code、GitHub Copilotなど）の特性を知りたい場合（本記事はAntigravity専用）

- **Decision Triggers**:
  - AIが生成したコードが「動くけど、これじゃない」と感じたとき
  - AIへの指示が曖昧で、期待する成果物が得られないとき
  - テストコードやエラーハンドリングがAI生成コードに含まれないとき
  - 自分の「エンジニアとしての常識」をAIに伝える方法がわからないとき
  - AI駆動開発における要件定義の重要性を再認識したいとき
