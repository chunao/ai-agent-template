# 【Antigravity】知らんかったの、わいだけ！？暴走を防ぐ「共通ルール」設定が超重要だった件

## TL;DR & Key Conclusions

Antigravityを使っていて「勝手にフォルダを作られる」「中身をぐちゃぐちゃに入れ替えられる」などの暴走問題に悩んでいる場合、**共通ルール（グローバルルール）**を設定することで解決できる。ルールを設定ファイルに記述しておけば、Antigravityはその通りに動いてくれる。

- **共通ルール設定が暴走防止の鍵**: Antigravityの予期しない動作を制御するには、事前にルールを設定しておくことが重要
- **設定場所**: `~/.gemini/GEMINI.md` ファイルにルールを記述する
- **安全管理ルールが特に重要**: ファイル操作やプログラム実行前に必ずユーザー確認を取るルールを設定することで、勝手な変更を防止できる
- **日本語対応**: 全て日本語で応答させるルールを設定可能
- **カスタマイズ推奨**: まずは既存のルール例を真似し、慣れたら自分専用にカスタマイズするのがベスト

## Quickstart

1. **設定ファイルを開く/作成する**: `~/.gemini/GEMINI.md` を開く（存在しなければ作成）
2. **基本ルールを記述する**: コミュニケーションルール（日本語で応答など）と安全管理ルール（操作前にy/n確認など）を記述
3. **保存して適用**: ファイルを保存すれば、Antigravityが次回起動時からルールに従う
4. **カスタマイズ**: 慣れてきたらAIに「私の仕事に最適なルールを作って」と相談して調整

## Context & Claims (Claim-Evidence-Caveat)

- **Claim**: 共通ルールを設定することでAntigravityの暴走を防げる
    - **Evidence**: 著者自身がルール設定により「勝手にフォルダを作られる」「中身を入れ替えられる」問題が解消したと報告
    - **Caveat**: ルールの記述内容によっては期待通りに動作しない可能性がある

- **Claim**: ファイル操作前のy/n確認ルールが安全管理に有効
    - **Evidence**: 「ファイルの作成・変更・削除、およびプログラムの実行を行う前に、必ず作業計画を報告し、『y/n』でユーザー確認を取り、『y』が返るまで一切の実行を停止する」というルール例を提示
    - **Caveat**: 毎回確認が入るため、単純作業の効率は低下する可能性がある

- **Claim**: 代替案の独自実行を禁止するルールが重要
    - **Evidence**: 「当初の実装方法で問題が発生した場合、独自の判断で代替案を実行せず、必ずユーザーに説明し、承認を得てから進める」というルール例
    - **Caveat**: トラブルシューティングに時間がかかる可能性がある

## Normalized Conditions

- **Use Cases**:
    - Antigravityを使っていて勝手にファイルやフォルダを変更される問題に困っている場合
    - AIエージェントの動作を制御・制限したい場合
    - 日本語での対話を強制したい場合
    - 操作前に必ず確認を入れたい場合

- **Anti-Cases**:
    - 完全自動化を求めており、確認なしで高速に処理を進めたい場合
    - Antigravity以外のAIツールを使用している場合（設定方法が異なる）

- **Prerequisites**:
    - Antigravityがインストールされていること
    - `~/.gemini/GEMINI.md` ファイルへのアクセス権限

- **Decision Triggers**:
    - Antigravityが勝手にフォルダを作成するようになった
    - 意図しないファイル変更が発生した
    - AIの応答が英語になって困っている

- **Failure Modes / Risks**:
    - ルールが曖昧だとAIが解釈を誤る可能性
    - 過度に制限的なルールは作業効率を下げる
    - ルールファイルの場所や形式を間違えると適用されない

- **Operational Cost**:
    - 初期設定の手間（数分程度）
    - y/n確認による作業フローへの割り込み
