# everything-claude-codeの設計思想

## TL;DR & Key Conclusions
「everything-claude-code」はAnthropicハッカソン優勝者が公開したClaude Code設定集。本質は「コンテキストを最適化する」という考え方であり、専門エージェントへの役割委譲、ルールの明文化、自動化、リソース効率管理を通じてClaude Codeの能力を最大化する。

- **コンテキストエンジニアリングが基盤**: 情報の精選がすべての施策に共通する原理
- **専門家チームを構成**: 単一AIではなく、特定タスクに特化したエージェントに役割委譲
- **マニュアルの明文化**: 暗黙的なルールを文書化し、毎回の説明の手間を削減
- **自動化で確実性確保**: AIの判断に頼らず機械的に実行する仕組みで漏れを防ぐ
- **リソース効率管理**: コンテキストウィンドウ保護のため、必要なツール以外は無効化
- **MCP過剰有効化の警告**: MCPを過度に有効化すると200kあったコンテキストが70kまで減少する可能性

## Quickstart
1. **段階的導入**: console.log警告フックのような単純な機能から開始
2. **MCP絞り込み**: 設定時に20〜30個登録しても、プロジェクトごとに10個以下に絞る
3. **ツール数制限**: 有効なツールは80個以下に抑える

## Context & Claims (Claim-Evidence-Caveat)
- **Claim**: コンテキスト最適化がClaude Code活用の本質
    - **Evidence**: Anthropicハッカソン優勝者の設定集がこの原理に基づいて設計されている
    - **Caveat**: 最適化の度合いはプロジェクトの規模や性質によって調整が必要

- **Claim**: MCPの過剰有効化はコンテキストを大幅に消費する
    - **Evidence**: 200kあったコンテキストが70kまで減少する可能性があると警告
    - **Caveat**: 必要なMCPは有効にすべきであり、完全に無効化するのは本末転倒

- **Claim**: 専門エージェントへの役割委譲が効率的
    - **Evidence**: 会社組織のように agents=専門部署、skills=業務マニュアル、rules=社訓と役割分担
    - **Caveat**: エージェント間の連携コストが発生する可能性

## Normalized Conditions
- **Use Cases**:
    - Claude Codeの設定を体系的に整理したい場合
    - コンテキストウィンドウを効率的に活用したい場合
    - 複数のエージェントやスキルを組み合わせた開発ワークフロー構築
    - チームでClaude Codeの設定を共有・標準化したい場合
- **Anti-Cases**:
    - 単純なタスクでClaude Codeを使う場合
    - 設定のカスタマイズに時間をかけられない場合
    - 少数のMCPで十分な小規模プロジェクト
- **Prerequisites**:
    - Claude Code CLI
    - agents, skills, hooks, rulesなどの基本概念の理解
    - MCPの基本知識
- **Decision Triggers**:
    - コンテキストウィンドウの枯渇を経験したとき
    - Claude Codeの設定が複雑化してきたとき
    - 「毎回同じ説明をしている」と感じたとき
    - Anthropicハッカソン優勝者のベストプラクティスを取り入れたいとき
