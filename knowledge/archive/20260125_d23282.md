# Claude Codeのカスタムスラッシュコマンド統合について

## TL;DR & Key Conclusions

Claude Codeのカスタムスラッシュコマンド機能がスキルシステムに統合され、`.claude/commands/` と `.claude/skills/` の両形式が同等に扱われるようになりました。既存の commands 形式は引き続きサポートされ、後方互換性が保たれています。

- **統合後も旧形式は動作**: `.claude/commands/xxx.md` 形式のカスタムコマンドは引き続き使用可能
- **同等の扱い**: commands と skills の両形式が `/skills` コマンドで一覧表示される
- **優先順位**: 同名のファイルが両方存在する場合、スキル形式（`.claude/skills/`）が優先される
- **自動実行の制御**: フロントマターに `disable-model-invocation: true` を追加することで、明示的な呼び出し時のみ実行されるように設定可能
- **使い分けの推奨**: 単発のヘルパーや既存コマンドは旧形式を継続、サポートファイルが必要な複雑な構成は新形式を使用

## Quickstart

1. **既存のコマンドの確認**: `.claude/commands/` ディレクトリ内のカスタムコマンドはそのまま動作し続けます
2. **スキル形式への移行（任意）**: 複雑な構成やサポートファイルが必要な場合は `.claude/skills/xxx/SKILL.md` 形式に移行を検討
3. **自動実行の無効化**: コマンドのフロントマターに `disable-model-invocation: true` を追加して、明示的な呼び出し時のみ実行されるように設定
4. **確認**: `/skills` コマンドで両形式のコマンド/スキルが表示されることを確認

## Context & Claims (Claim-Evidence-Caveat)

- **Claim**: カスタムスラッシュコマンドとスキルが統合された
    - **Evidence**: 公式ドキュメント（2026年1月時点）に「カスタムスラッシュコマンドはスキルにマージされました」と記載
    - **Caveat**: 統合後も `.claude/commands/` 形式は引き続きサポートされ、後方互換性が維持されている

- **Claim**: 同名ファイルが存在する場合、スキル形式が優先される
    - **Evidence**: 記事内で「同じ名前のファイルが両方存在する場合、スキル形式が優先されます」と明記
    - **Caveat**: 意図しない優先順位の問題を避けるため、同名ファイルの重複は避けるべき

- **Claim**: `disable-model-invocation: true` で自動実行を防げる
    - **Evidence**: フロントマターにこの設定を追加することで、ユーザーが明示的にコマンドを実行したときのみ動作する
    - **Caveat**: この設定がない場合、Claude が自動的にコマンドを呼び出す可能性がある

## Code Snippets

- **自動実行無効化設定**: コマンドファイルのフロントマターに追加することで、明示的な呼び出し時のみ実行されるように制御

```yaml
---
disable-model-invocation: true
---
```

## Normalized Conditions

- **Use Cases**:
    - 既存の `.claude/commands/` 形式のカスタムコマンドを継続使用したい場合
    - 単発のヘルパーコマンドやシンプルなスラッシュコマンドを維持したい場合
    - サポートファイル（スクリプト、設定ファイル等）が必要な複雑なスキルを新形式で構築したい場合
    - Claude の自動実行を防ぎ、明示的な呼び出しのみで動作させたい場合

- **Anti-Cases**:
    - 特になし（両形式がサポートされているため、使い分けは柔軟）

- **Decision Triggers**:
    - 複雑な構成やサポートファイルが必要になった時点で、スキル形式への移行を検討
    - 自動実行による意図しない動作が発生した場合、`disable-model-invocation: true` の追加を検討
    - 同名のコマンド/スキルが存在し、優先順位の問題が発生した場合、どちらか一方に統一
