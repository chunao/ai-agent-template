# GPT-5.2 Prompting Guide

## TL;DR & Key Conclusions
GPT-5.2向けの本番運用を想定したプロンプティング戦略ガイド。エンタープライズやエージェントワークフローでの性能最大化を目的とし、GPT-5.2固有の振る舞いに対応した手法を解説。明示的・構造化されたプロンプトが特に効果的で、スコープ境界、出力制約、推論期待値の定義が重要。

- **より慎重なスキャフォールディング**: GPT-5.2は中間構造を自然に構築する傾向がある
- **低冗長性**: 簡潔な出力を生成しつつ、プロンプトに応じた調整が可能
- **強い指示遵守**: ユーザー意図からのドリフトが減少
- **保守的な根拠バイアス**: 明示的な推論と正確性を重視
- **長文コンテキスト対策**: 10kトークン超では内部要約を強制すべき
- **ツール最適化**: 独立操作の並列化と高影響変更の検証ステップが重要
- **構造化抽出**: 明示的なスキーマやJSON形式の指定が必須
- **マイグレーション時の注意**: モデル変更とプロンプト変更は分離してテスト

## Quickstart
1. **冗長性制御の設定**: 「デフォルト: 3〜6文または5箇条書き以下」のような具体的な長さ制約を設定
2. **スコープ規律の適用**: 「ユーザーが要求したことのみを正確に実装。追加機能、追加コンポーネント禁止」を明記
3. **曖昧性管理の導入**: 詳細不足の質問には、ラベル付きの仮定と共に解釈候補を提示させる
4. **ツール説明の最適化**: 1〜2文以内に収める
5. **GPT-4oからの移行時**: `reasoning_effort`を"none"に設定
6. **GPT-5.1からの移行時**: 既存の effort レベルを維持

## Context & Claims (Claim-Evidence-Caveat)
- **Claim**: GPT-5.2は明示的で構造化されたプロンプトで最も効果を発揮する
    - **Evidence**: スコープ境界、出力制約、推論期待値を定義することで、本番システムでの信頼性と一貫性が向上
    - **Caveat**: プロンプトの最適化は段階的に行い、1サイクルにつき1つの制約のみ調整すべき

- **Claim**: 長文コンテキスト（10kトークン超）では「スクロール内迷子」エラーが発生する
    - **Evidence**: 内部要約を強制することでユーザー制約の再確認が行われ、エラーが減少
    - **Caveat**: すべてのドキュメントに適用する必要はなく、10kトークン超の場合のみ

- **Claim**: GPT-5.2はツール使用の最適化が可能
    - **Evidence**: 独立操作の並列化と高影響変更の検証ステップにより効率向上
    - **Caveat**: ツール説明は1〜2文に収める必要がある

- **Claim**: 構造化抽出には明示的なスキーマが必要
    - **Evidence**: 「ソースにフィールドが存在しない場合は推測せずnullを設定」という指針
    - **Caveat**: スキーマ定義の手間が追加で発生する

- **Claim**: GPT-5.2 with Reasoningはコンパクション機能をサポート
    - **Evidence**: `/responses/compact`エンドポイントで暗号化された状態保存が可能、コンテキストウィンドウ制限を超えた継続が可能
    - **Caveat**: マルチステップエージェントフローに特に有効だが、単純なタスクには不要

- **Claim**: モデル移行時は変更を分離すべき
    - **Evidence**: モデル変更とプロンプト変更を同時に行うと問題の切り分けが困難になる
    - **Caveat**: 評価でベースラインを計測してから最適化を行うこと

## Normalized Conditions
- **Use Cases**:
    - エンタープライズ向けLLMアプリケーション開発
    - エージェントワークフローの構築
    - 長文ドキュメント処理システム
    - 構造化データ抽出パイプライン
    - GPT-4oまたはGPT-5.1からのマイグレーション
    - Web検索を活用したリサーチタスク
- **Anti-Cases**:
    - シンプルな一問一答型のチャットボット
    - プロンプト最適化の余地がない単発タスク
    - GPT-5.2以外のモデルを使用する場合
- **Prerequisites**:
    - OpenAI APIへのアクセス
    - GPT-5.2モデルの利用権限
    - 基本的なプロンプトエンジニアリングの知識
- **Decision Triggers**:
    - GPT-5.2を本番環境で使用する予定がある
    - 既存のGPT-4o/GPT-5.1からの移行を検討中
    - LLM出力の一貫性・信頼性向上が課題
    - エージェント型ワークフローの構築を検討中
