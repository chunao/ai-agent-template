# [Claude Code] AIコーディング開発サイクル (2026年1月時点)

## TL;DR & Key Conclusions

2026年1月時点での個人開発におけるAIコーディングツール活用のワークフローを5つのフェーズ（Plan, Implementation, Review, Resolve Review, Quality Assurance）で体系化した実践ガイド。前回（2025年11月）からの2ヶ月間のアップデート内容を含む。

- **AIだけに頼らない品質保証**: CLAUDE.mdなどの指示は忘れられたり解釈が曖昧になるため、カスタムリント・スクリプト・静的解析と組み合わせることで一貫性のあるコード品質を実現する
- **Sandbox環境への移行**: Devcontainer（Docker）からSandbox環境へ変更し、軽量化・資源制限の撤廃・プロジェクト毎の設定分離を実現
- **多層的なレビュー体制**: GitHub Copilot、Claude Code、Gemini Review、Codexを活用。ChatGPT Plus（月20ドル）のレビュー機能がコスパ最高と評価
- **レビューコメント自動解決**: `/resolve-all-gh-review-comments`でOutdatedを自動解決、Validなコメントをグループ化して効率化
- **具体的なチェックリスト化**: 抽象的な指摘（"Clean Architectureに従っているか"）より、白黒つけやすい具体的な項目（依存関係方向性、レイヤー間参照制限など）に変換することを推奨
- **Phase 1（Plan）の重要性**: GitHub Issueで現状分析・方針決定を行い、AIに関連コード調査・ベストプラクティス収集・根本原因特定を依頼
- **Phase 2の柔軟な実装方法**: Slash Command（`/resolve-gh-issue`）での自動化と、リモート環境（Slack経由でCodex/Claude呼び出し）での細かい修正の使い分け
- **カスタムリント化の推進**: AIレビューで頻出の指摘（例：Domain層がInfrastructure層をimport）をスクリプト化し、pre-commitフックで自動実行

## Quickstart

### 1. **Sandbox環境のセットアップ**
`~/.claude/settings.json`に以下を設定：

```json
{
  "sandbox": {
    "enabled": true,
    "network": {
      "allowedDomains": ["github.com", "npmjs.org"]
    },
    "excludedCommands": ["git:*", "gh:*", "pre-commit:*"]
  }
}
```

### 2. **Phase 1: Plan**
GitHub Issueを作成し、AIに以下を依頼：
- 関連コードの調査
- ベストプラクティスの収集
- 根本原因の特定

### 3. **Phase 2: Implementation**
- **方法1（自動化）**: `/resolve-gh-issue #111`でPR自動作成
- **方法2（細かい修正）**: Slack経由でCodex/Claude Codeをリモート呼び出し

### 4. **Phase 3: Review**
複数のAIツールでレビュー：
- GitHub Copilot
- Claude Code
- Gemini Review
- Codex
- ChatGPT Plus（コスパ最高）

### 5. **Phase 4: Resolve Review**
`/resolve-all-gh-review-comments <PR番号>`で自動分類・解決

### 6. **Phase 5: 品質保証**
カスタムリントスクリプトをpre-commitフックで自動実行

## Context & Claims (Claim-Evidence-Caveat)

- **Claim**: AIツールの指示（CLAUDE.md等）は忘れられたり解釈が曖昧になる
    - **Evidence**: 筆者の実践経験。指示ファイルだけでは一貫性のあるコード品質を保てない
    - **Caveat**: カスタムリント・スクリプト・静的解析と組み合わせることで解決可能

- **Claim**: Sandbox環境はDevcontainerより優れている
    - **Evidence**: Dockerレスで軽量、資源制限なし、プロジェクト毎の設定を分離可能
    - **Caveat**: 2026年1月時点での評価。将来的な変更の可能性あり

- **Claim**: ChatGPT Plus（月20ドル）がレビューツールとしてコスパ最高
    - **Evidence**: 筆者の使用経験に基づく主観的評価
    - **Caveat**: 他のAIツールとの組み合わせが前提

- **Claim**: 抽象的なチェック項目より具体的なチェックリストが有効
    - **Evidence**: "Clean Architectureに従っているか"より"Domain層がInfrastructure層をimportしていないか"の方が白黒つけやすい
    - **Caveat**: プロジェクトの性質により適切な粒度は異なる

- **Claim**: レビューコメントの自動分類・解決でレビュー効率が向上
    - **Evidence**: `/resolve-all-gh-review-comments`でOutdated自動解決、Validなコメントをグループ化
    - **Caveat**: 完全自動化ではなく、人間の最終判断が必要

## Code Snippets

- **Sandbox設定**: `~/.claude/settings.json`でネットワーク制限・除外コマンドを設定し、安全性とプロジェクト分離を実現
- **カスタムリントスクリプト**: Domain層とInfrastructure層の依存関係チェックなど、AIレビューで頻出の指摘を自動化
- **Slash Command活用**: `/resolve-gh-issue #111`でIssueからPRまで自動化、`/resolve-all-gh-review-comments <PR番号>`でレビューコメント自動分類

## Normalized Conditions

- **Use Cases**:
    - 個人開発でAIコーディングツールを活用したい場合
    - 複数のAIツール（Claude Code、GitHub Copilot、Codex、Gemini等）を組み合わせて品質を向上させたい場合
    - レビュープロセスを効率化・自動化したい場合
    - DevcontainerからSandbox環境への移行を検討している場合
    - AIレビューの指摘を自動的に強制したい場合（カスタムリント化）

- **Anti-Cases**:
    - チーム開発で統一的なワークフローが必要な場合（個人開発向けの内容）
    - AIツールを使用しない開発環境
    - 細かいカスタマイズやスクリプト作成が不要な簡易プロジェクト

- **Decision Triggers**:
    - AIコーディングツールの活用方法を体系化したい
    - コードレビューの品質と効率を同時に向上させたい
    - AIの指示が守られない・解釈が曖昧になる問題に直面している
    - Devcontainer（Docker）の重さや資源制限に不満がある
    - レビューコメントの手動対応に時間がかかりすぎている
