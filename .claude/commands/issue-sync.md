# /issue-sync - Issue進捗同期

GitHub Issueに現在の進捗を同期します。

## 使い方

```
/issue-sync 123
```

## 実行手順

### 1. 現在の状態を確認

以下を確認してください：

- 現在のブランチ
- コミット履歴（Issue開始以降）
- 変更されたファイル
- テストの実行結果

```bash
git log --oneline -10
git diff --stat HEAD~5
```

### 2. 進捗レポートの作成

以下の形式で進捗をまとめてください：

```markdown
## 進捗報告 - {現在日時}

### 完了したタスク
- [x] {完了したタスク1}
- [x] {完了したタスク2}

### 進行中のタスク
- [ ] {進行中のタスク} - {進捗率}%

### 残タスク
- [ ] {残りのタスク1}
- [ ] {残りのタスク2}

### 実装内容
- {変更したファイル}: {変更内容}

### テスト状況
- 実行: {成功数}/{全体数}
- カバレッジ: {X}%

### ブロッカー（あれば）
- {問題点や相談事項}

### 次のアクション
1. {次にやること}
```

### 3. Issueコメントに投稿

作成した進捗レポートをGitHub Issue #$ARGUMENTS のコメントとして投稿してください。

```bash
gh issue comment $ARGUMENTS --body "進捗レポートの内容"
```

### 4. チェックリストの更新

Issueの本文または以前のコメントにあるチェックリストを更新：

- 完了したタスクは `[x]` に変更
- 新たに判明したタスクは追加

## 同期タイミングの目安

| 状況 | 推奨頻度 |
|------|---------|
| 順調に進行中 | 1日1回 |
| 複雑なタスク | タスク完了ごと |
| ブロッカー発生 | 即時 |
| レビュー前 | 必須 |

## 出力例

```
## Issue #$ARGUMENTS 同期完了

**最終更新**: {日時}
**進捗**: {完了タスク数}/{全タスク数} ({進捗率}%)
**ステータス**: {順調 / 遅延 / ブロック中}

Issueコメントを更新しました。
→ https://github.com/{owner}/{repo}/issues/$ARGUMENTS
```

## 関連コマンド

- `/start-issue $ARGUMENTS` - Issue作業を開始
- `/review` - コードレビューを実行
- `/tdd` - TDDサイクルを継続
