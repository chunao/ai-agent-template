# /plan - 実装計画

タスクの実装計画を立てます。

## 手順

### 1. 要件の確認

- 何を実装するのか明確にする
- 受け入れ基準を定義する
- 技術的制約を洗い出す

### 2. 設計案の検討

3つの設計案を検討してください：

```markdown
## 案1: [アプローチ名]
- 概要:
- メリット:
- デメリット:
- 評価点: /100

## 案2: [アプローチ名]
- 概要:
- メリット:
- デメリット:
- 評価点: /100

## 案3: [アプローチ名]
- 概要:
- メリット:
- デメリット:
- 評価点: /100
```

### 3. 推奨案の選定

- 80点以上の案のみ採用
- 選定理由を明記

### 4. タスクのブレイクダウン

チェックリスト形式で分解：

```markdown
## 実装タスク

- [ ] タスク1
- [ ] タスク2
- [ ] タスク3
```

## 注意事項

- 設計はGitHub Issueコメントに記録（ローカルMDは残さない）
- タスクは小さな単位に分割してレビューしやすくする
