# コーディングスタイルルール

以下のコーディングスタイルを遵守してください。

## 命名規則

### 一般原則

- 意味のある名前を使用する
- 略語は避ける（一般的なものは除く）
- 一貫性を保つ

### 言語別ガイドライン

#### Python
- 変数・関数: `snake_case`
- クラス: `PascalCase`
- 定数: `UPPER_SNAKE_CASE`
- プライベート: `_prefix`

#### JavaScript/TypeScript
- 変数・関数: `camelCase`
- クラス: `PascalCase`
- 定数: `UPPER_SNAKE_CASE`
- コンポーネント: `PascalCase`

## コード構造

### DRY原則

- 同じコードを3回以上書かない
- 共通処理は関数・メソッドに抽出する

### 単一責任の原則

- 1つの関数は1つのことだけを行う
- 1つのクラスは1つの責任だけを持つ

### 関数の長さ

- 1関数は50行以内を目安にする
- 長くなる場合は分割を検討する

## コメント

- 「なぜ」を説明するコメントを書く
- 「何を」しているかはコードで表現する
- 古くなったコメントは削除または更新する
