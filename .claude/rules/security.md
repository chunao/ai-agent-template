# セキュリティルール

以下のセキュリティルールを常に遵守してください。

## 必須事項

### シークレット管理

- APIキー、パスワード、トークン等をコードにハードコードしない
- 環境変数または設定ファイル（.env）を使用する
- .envファイルは.gitignoreに含める

### 入力バリデーション

- ユーザー入力は必ずバリデーションする
- SQLクエリにはパラメータ化クエリを使用する
- HTMLエスケープを適切に行う（XSS対策）

### 認証・認可

- パスワードは必ずハッシュ化して保存する
- セッション管理を適切に行う
- 権限チェックを怠らない

## 禁止事項

- `eval()`や動的コード実行の使用
- ユーザー入力をそのままシェルコマンドに渡す
- 本番環境でデバッグモードを有効にする
- エラーメッセージでシステム内部情報を露出する
